Several parameters may be passed to the URL to select different kinds of reports.
The URLs associated with the above four links are:

"http://stark-retreat-1229.herokuapp.com/meds/printform?formtype=hs-master-schedule&day=wed&dest=bvsatow@hpeprint.com"

"http://stark-retreat-1229.herokuapp.com/meds/printform?formtype=caregroup-med-audit-sheets&dest=bvsatow@hpeprint.com"

"http://stark-retreat-1229.herokuapp.com/meds/printform?formtype=caregroup-med-audit-sheets&camperid=548&dest=bvsatow@hpeprint.com"

"http://stark-retreat-1229.herokuapp.com/meds/printform?formtype=all&camperid=548&dest=bvsatow@hpeprint.com"

The preamble to the http should contain credentials to avoid having to enter password.

The parameters specified in the URL are:
  • formtype
    This parameter indicates which of the seven forms to select.
    Options:
    • formtype=all
    • formtype=hs-master-schedule
    • formtype=hs-med-audit-sheets
    • formtype=caregroup-hs-med-schedule
    • formtype=bus-med-audit-sheet
    • formtype=caregroup-med-audit-sheets
    • formtype=guardian-signature-sheet
    • formtype=leftover-meds-receipt

    If not specifed, the default is:
    • formtype=all

  • year
    This parameter is the year of the Special Camp database.
    If not specifed, the default is:
    • year=2014

  • camperid
    This parameter is the id number of a camper.
    If not specifed, the default is:
    • camperid=all
    • camperid={some camper ID}

  • day
    This parameter indicates the day of the week.
    Options:
    • day=all
    • day=sun
    • day=mon
    • day=tue
    • day=wed
    • day=thu
    • day=fri
    • day=sat
    • day=asneeded

    If not specifed, the default is:
    • day=all

  • page
    Some of the responses consist of multiple pages. For example, the HS Master Schedule for Tuesday consists of 7 pages. This parameter specifies a particular page number to print.

    If not specifed, the default is:
    • page=all

  • refresh
    If the database has been changed it is necessary to load the new database items from the Quickbase website. This takes a few seconds. This parameter indicates the conditions for reloading the database.

    Options:
    • refresh=true unconditionally load the Quickbase Med Schedule Items Table
    • refresh=false do not load the Quickbase Med Schedule Items Table
    • refresh=check if the timestamps have changed on any of the 3 Quickbase tables then load the Quickbase Med Schedule Items Table otherwise do not load the Quickbase Med Schedule Items Table

    If not specifed, the default is:
    • refresh=check

  • dest
    This parameter is an email address indicating where the PDF file will be sent when the user clicks on "PRINT REMOTELY". The "dest" field may be set to "dest=default" and the administrator may change the default by invoking a web service request "setdefault". The default will hold until changed by sending a new "setdefault". The default will not be used if a particular dest is specified in the "dest" field of the "printform".
    Options:
    • dest=default
    • dest={some email address}

    If not specifed, the default is:
    • dest=default

    Here are examples of the format of the "setdefault" request:

    http://stark-retreat-1229.herokuapp.com/meds/setdefault?dest=kurt.wegner@gmail.com
    http://stark-retreat-1229.herokuapp.com/meds/setdefault?dest=bvsatow@hpeprint.com
